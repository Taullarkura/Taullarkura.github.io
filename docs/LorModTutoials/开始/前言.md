# 前言
由于现在网上成体系的图书馆mod教程基本没有，所以我决定自己写一份。

让人学习完这篇教程之后，能独立编写出废墟图书馆mod，这是本教程的基本目的。

因为月亮计划为图书馆提供的模组编辑器功能有限且受到阉割，所以本教程基于**Basemod**框架编写，但是本教程的大部分也适用于使用模组编辑器编写mod。

**无论如何，我仍然推荐你使用Basemod。**

在你阅读本教程之前，你至少需要有如下的知识与素养：
- <span style="color:red;font-weight: bold;">一定的C#代码基础</span>
- 能够通过阅读C#源代码理解程序逻辑
- xml编写基础

你可以通过[菜鸟教程](https://www.runoob.com/csharp/csharp-tutorial.html)、b站大学等方式学习它们。

### 本教程的结构

本教程分为四个部分：
- **开始** - 主要是制作模组前的准备部分，也包括一些你需要了解的内容
- **编写模组内容** - 具体的模组内容编写，如核心书页、战斗书页等
- **数据库** - 各种常用类的说明，如BattleUnitModel等
- **实践** - 部分常用(或不常用?)的具体效果示例，如**变幻莫测**。

### 关于代码
和其他Unity游戏一样，废墟图书馆使用C#编写，这意味着C#编程相关的能力在mod开发方面至关重要。

如果你有C++/Java基础(或者其他面向对象语言)，你可能会更能够理解C#，这种情况下我仍推荐先去学习C#的基础语法。

<span style="color:red;font-weight: bold;">本教程不会讲解C#语法方面的内容！</span>

同时，图书馆使用xml存储游戏中的大部分数据和本地化文本，所以你也需要具备xml编写能力。所幸，xml并不复杂，本教程也会提供一个简单的介绍和说明。

由于图书馆采用**面向接口编程**，图书馆的基础代码难度并不复杂，但是对于一些高级效果的编写，你可能会需要对Unity相关内容的了解，以及对Harmony Patch的掌握。

不用担心，不会就学就是了。
